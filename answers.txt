What does fast-forward in Git mean?
If the master is not diverged from the new branch created and instead of commiting if we can just move the pointer to the latest commit then it is called fast forwarding

Why should you have different branches?
By having different branches we can have multiple persons working on the same project and when they are done they can just push it to the master branch, by this we can avoid conflicts of multiple copies of the same file
